# Архитектура сайта «Тихий Шум»

Проект разделён по ответственности и следует правилу зависимостей: внешний слой может знать о внутреннем, внутренний — никогда о внешнем.

```text
presentation ─────► application ─────► domain
      ▲                    ▲
      │                    │ ports
      │            infrastructure
      └──────── app/compositionRoot.js
```

## Слои

- `domain` — заказ и его правила: позиции, количество, удаление, итоги. Здесь нет DOM, `localStorage` и сетевых API.
- `application` — сценарии пользователя (`GetMenu`, `AddItemToOrder`, `ChangeOrderQuantity`, `SubmitPreorder`) и интерфейсы репозиториев.
- `infrastructure` — конкретные реализации портов: каталог в памяти и сохранение заказа в `localStorage` с безопасным fallback в память.
- `presentation` — компоненты страницы, presenter-функции, контроллер взаимодействий и дизайн-система.
- `app` — composition root. Только здесь реализации репозиториев связываются со сценариями и UI.
- `shared` — неизменяемая конфигурация демонстрационного меню.

## Зачем это нужно

Чтобы подключить реальный backend, достаточно реализовать новые `CatalogRepository` и `OrderRepository`, затем заменить две строки в `compositionRoot.js`. Домен, сценарии заказа и интерфейс останутся без изменений.

## Границы демонстрации

Сайт не отправляет данные во внешние системы. Предзаказ имитируется локально и очищает корзину после успешного подтверждения. Адрес, часы работы, меню и цены — демонстрационные.
